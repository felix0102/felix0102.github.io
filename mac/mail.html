# 避免 Mac 邮件正文变成 HTML 附件的问题

# 问题描述

在使用 Mac 自带的邮件客户端发送邮件时，如果邮件包含附件， 附件后面的正文会被截掉，并形成一个 HTML 附件。 名字类似是 ATT00004.html 附件后面的正文被截掉了，并自动形成了 ATT00004.html（其中包含了后半部分正文）！

# 问题原因

问题原因 Mac 中编写邮件时，会将附件与邮件正文混编。然后发送这样混编的邮件至服务器。

而有些邮件服务器则坚持附件必须位于正文之后。 因此出现在附件之后的正文也会被当做附件对待，因此生成了额外的附件来包含后半部分正文，这就是上面的 AT00004.html。

# 避免这种情况出现的解决办法

  • 解决办法：在用mail发邮件时，不要带附件，如果要带附件，请将附件放到邮件的结尾处。
  • 辅助解决办法：在 Mail（13.4）中，点击菜单栏『编辑』->『附件』，勾选『始终在邮件结尾处插入附件』。这样你在邮件中插入附件时，附件会自动插入在邮件的结尾处。 注意，此处的『始终发送与 Windows 兼容的附件』也要勾选，否则很多邮件服务器都无法识别邮件附件。
